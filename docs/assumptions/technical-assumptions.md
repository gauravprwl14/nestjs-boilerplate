# Technical Assumptions

These are the assumptions baked into the current design.
If any assumption is invalidated, update this document and review the affected components.

---

## Runtime & Platform

| Assumption | Impact if wrong |
|-----------|----------------|
| Node.js 22 LTS is the runtime | OTel SDK and Prisma versions may need updating for older Node versions |
| Linux containers for production | Native bindings (none currently) may behave differently on Windows dev — use Docker for parity |
| Single-region deployment for this build | Multi-region adds clock drift concerns and Postgres replication complexity |

## Database

| Assumption | Impact if wrong |
|-----------|----------------|
| PostgreSQL 16 is the database | Prisma adapter swap needed for MySQL/SQLite; composite FKs and recursive CTEs are PG-portable but syntax differs |
| `@prisma/adapter-pg` native driver is used | Changing adapters may change connection pooling behaviour |
| All UUIDs generated by PostgreSQL `uuid()` default | Application-generated UUIDs would require schema migration |
| `@@unique([id, companyId])` on `Department` and `Tweet` is essential | Without it, the same-tenant composite FKs on pivots/parent cannot form |
| A single database schema serves all tenants | Per-tenant schemas would require major refactoring of `PrismaService` and all repos |

## Authentication

| Assumption | Impact if wrong |
|-----------|----------------|
| Mock auth via `x-user-id` is acceptable for this take-home | Swap `MockAuthMiddleware` for a real JWT/OIDC guard; it must publish the same CLS tuple (`USER_ID`, `COMPANY_ID`, `USER_DEPARTMENT_IDS`) to keep everything downstream unchanged |
| The header contains a server-issued UUID (seed-printed) | Real deployments must ensure the header is not user-forgeable — current middleware trusts it completely |
| Email uniqueness is enforced at DB level | If that constraint is removed, ambiguous lookups become possible |

## Tenancy

| Assumption | Impact if wrong |
|-----------|----------------|
| CLS (AsyncLocalStorage) propagates tenant context to every await | A misconfigured module or a non-async-aware library could lose context — the `AuthContextGuard` is the fail-fast backstop |
| The Prisma `$extends` extension is the primary ORM-layer defense | Extensions do NOT intercept raw SQL or nested `connect` — services are responsible for those cases and the schema composite FKs back them up |
| `Company` and `User` are NOT tenant-scoped | `User` lookup must complete before tenant context exists; `Company` IS the tenant record |

## Infrastructure

| Assumption | Impact if wrong |
|-----------|----------------|
| OTel Collector is optional (`OTEL_ENABLED=false` by default) | Disabling OTel removes distributed tracing; logs still work via Pino |
| No Redis / no BullMQ | No async-job dispatch available — reintroduce `QueueModule` if needed |
| Docker Compose is sufficient for local dev | Teams using Kubernetes locally will need to adapt compose to helm |

## Security

| Assumption | Impact if wrong |
|-----------|----------------|
| HTTPS termination happens at the load balancer/proxy, not in the app | Running the app directly on HTTPS requires adding TLS config to `main.ts` |
| `x-user-id` mock auth is REPLACED before production | Shipping mock auth to production is a total auth bypass — see the archived `2026-04-17_FOR-Authentication.md` for the previous JWT/API-key stack as a reference implementation |

## Testing

| Assumption | Impact if wrong |
|-----------|----------------|
| Integration tests run against a real PostgreSQL test database | Using SQLite in-memory for tests would miss composite-FK and recursive-CTE behaviour |
| ≥ 70% global line coverage, ≥ 80% on services is the minimum | Lower coverage may leave critical error paths untested |
| The 13-case ACL matrix is the source of truth for visibility correctness | Changing the timeline SQL without updating `test/integration/acl-matrix.spec.ts` will silently break visibility |
