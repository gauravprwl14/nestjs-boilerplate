# docs/CONTEXT.md — Layer 2 Router

Route your question to the correct subfolder.

| Question | Subfolder |
|----------|-----------|
| What are the product requirements and API endpoints? | `docs/prd/` |
| How is the system architected at a high level? | `docs/architecture/` |
| How do the flows work (auth, CRUD, errors, observability)? | `docs/diagrams/` |
| What coding patterns and standards do we follow? | `docs/coding-guidelines/` |
| How is infrastructure set up (Docker, env vars, deploy)? | `docs/infrastructure/` |
| What are the feature guides (deep-dives per feature)? | `docs/guides/` |
| What is the current plan and planning template? | `docs/plans/` |
| What assumptions underpin the technical design? | `docs/assumptions/` |
| What tasks are done and what is pending? | `docs/task-tracker/` |
| Where is the generated API spec? | `docs/api/` |

Each subfolder contains its own `CONTEXT.md` for further routing.
For system-wide orientation start at `CLAUDE.md` in the project root.
